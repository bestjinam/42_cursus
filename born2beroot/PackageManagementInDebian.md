# 패키지와 패킷 
## 패키지
- 소프트웨어를 실행하는데 필요한 파일들(실행, 설정 라이브러리 등) 이 담겨 있는 설치 파일 묶음
- 소스코드가 담겨 있는 소스 패키지와 컴파일된 바이너리 파일이 담겨 있는 바이너리 패키지가 있음

### DEB
Debian 계열의 패키징 방식 

### RPM 
Redhat 계열 패키지 방식

## 리눅스 패키지 관리 방식 
### 저수준 패키지 도구 

패키지를 설치, 제거 하는 도구로, 패키지 간의 의존성은 알지 못함 
Deb -> dpkg

Rpm -> rpm

#### 고수준 패키지 도구 

패키지 파일 설치, 제거하는 기능 뿐 아니라 다운로드 가능한 패키지를 검색하고, 패키지간 의존성을 관리한다. 
한 패키지를 사용하기 위해 다른 패키지가 필요한 경우 다른 패키지 까지 설치 해줌 

deb -> apt-get, apt, aptitude

rpm -> yum, dnf

## 패키지 설치 과정 

패키지 관리자는 설치가능한 패키지 정보를 가진 메타 정보를 로컬 캐시에 저장한다. 

패키지 설치 명령이 오면 패키지 관리자는 로컬 캐시를 참조하여 패키지 정보를 찾아, 패키지 저장소에 요청 하여 패키지를 다운로드 한다. 

고수준 패키지 도구의 경우 패키지간 의존성을 확인하여 필요한 패키지를 모두 설치 한다. 



#  Debian 의 패키지 관리 도구 

## Apt 

고수준 패키지 도구로, apt-get, apt-cache, apt-config 등의 툴들을 모아 apt-get 보다 더 보기 좋고 편리하게 사용할 수 있도록 만든 도구이다. 
apt-get이 apt 보다 오래 되어 옵션들이 더 많지만, 잘 쓰이지 않는 옵션들도 많은데, apt는 이렇게 잘 쓰이지 않는 기능들을 제외 하였다. 

## aptitude

aptitude는 apt에 그래픽 인터페이스를 더한 좀 더 고차원적인 관리 도구이다.
apt 의 모든 기능을 지원하며, 설치와 제거시 생기는 충돌에 대한 관리를 지원해준다고 한다. 
또한 특정 패키지에 대해 why, why-not 옵션을 제공하여 어떤 패키지가 왜 필요한지/필요 없는지에 대해 물을 수 있다. 

# 새로운 패키지를 설치 하기 전에 항상 해야할일 

- apt update : 가능한 모든 패키지 정보를 업데이트 한다.  
- apt upgrade : 업데이트가 필요한 모든 패키지를 실제로 업데이트 한다.
